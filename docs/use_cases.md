# Use cases

## Цели (Objective)

### Создание цели пользователем

POST `/api/objectives/`
Пользователь создаёт цель определенного типа (ачивка, квест, вызов).
Заполняет:

- название
- описание
- количество, которое нужно выполнить
- начальное значение (опционально)
- сложность (1-4)
- родительская цель (опционально; должна быть такого же типа)
- частота выполнения (`frequency`), только для цели с типом `CHALLENGE` (вызов)

### Получение целей пользователя по типу и фильтрам

GET `/api/objectives/`
Пользователь передаёт тип цели и фильтры:

- скрыть выполненные

В ответе получает список целей одного типа.

### Изменение цели пользователем

PUT `/api/objectives/{id}`
Пользователь может изменить:

- название
- описание
- количество для выполнения (не может быть меньше, чем текущее значение)
- сложность (не может быть меньше чем у родительской ачивки)

### Увеличение счётчика цели

POST `/api/objectives/{id}/increase_counter`

Пользователь может увеличить счётчик цели своей.

На каждое изменение счётчика создаётся событие процесса с привязкой к цели.

За каждое увеличение начисляется экспа в прогресс пользователя по формуле:

- если цель ещё в процессе: `'сложность'`,
- если цель завершена: `'сложность' * 2`

Если текущей экспы достаточно для перехода на следующий уровень, то увеличиваем уровень.
Пороги экспы для уровней определяем по таблице `уровни`.

### Уменьшение счётчика цели

POST `/api/objectives/{id}/reduce_counter`

## События прогресса (ProgressEvent)

### Получить список событий пользователя

GET `/api/progress_events/{user_id}`
Пользователь может получить список событий
отсортированный в порядке создания от самых новых к самым старым с пагинацией

# Прогресс пользователя (UserProgress)

## Получение прогресса пользователя

GET `/api/user_progress/{user_id}`

Прогресс пользователя:

- текущий уровень
- экспа внутри уровня
- экспы нужно на текущий уровень
- прогресс уровня
- экспы до следующего уровня
